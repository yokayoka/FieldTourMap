# Field Tour

地理学・地質学の野外教育向けWebGISアプリ。スマートフォンで運用でき、GNSSによる現在地表示、Leafletベースの複数レイヤー切替、見学ポイント（POI）ごとの写真・動画・参考文献リンク表示、観察メモの記録、オフラインタイルキャッシュ、URL共有機能、複数実習ツアーの切替に対応する。GitHub Pagesの静的ホスティングのみで動作する。

- 本番環境: https://yokayoka.github.io/FieldTourMap/

## 作成の動機
私は巡検 (field tour)が大好です。私自身は地形学が専門ですが、学生や研究仲間とともにテーマを決めて、野外で議論をするのは本当に楽しいと思います。地形学の巡検に関しては航空レーザー測量による高解像度の標高データとそれから作成された詳細な各種の微地形図が公開されるようになり、現地の現象と微地形データを照合することで謎解きをすることの楽しさが、ますます深まりつつあります。ただし、巡検の準備はなかなか大変です。対象地域の地質図や地形図、関連する論文をコピーして人数分の資料を準備するのですが、現地に行ってからデータの不足を認識することもしばしばあります。
ところが、WebGISの技術が進展し、[Leaflet](https://leafletjs.com/)などの優れたライブラリーが利用可能になってからは、デジタル技術を巡検に活用できるようになりました。とくに、地図データに関しては[スーパー地形](https://play.google.com/store/apps/details?id=com.kashmir3d.superdem&hl=ja&pli=1)や[Avenza maps](https://avenzamaps.jp/)を導入することで、準備がずいぶんと楽になりました。これらのアプリは電波が届かないオフライン環境に対応しているのも嬉しいところです。それでも、これらのアプリに慣れない人もいるし、有料だったりするので、参加者全員に同じ環境を準備するのも、それなりに大変です。また、関連文献や露頭スケッチなどの紙の配布資料の準備は、これまで通りたくさんのコピー資料を準備する必要があります。このような、面倒な作業を楽にして、もっと気軽に巡検を楽しめるようにできないかと思ったのが、本アプリ作成の動機です。これには、素人の私が、稚拙ながらもこのようなアプリを作ることが出来たのは、ClaudeCodeのようなエージェント型AIを利用できるようになったためです。また、いろいろな地域のデータがオープンデータとして公開されたからこそ、それを組み合わせるアプリ側だけを準備すれば、コンテンツが出来る環境が実現したわけです。このような、OSSやオープンデータを推進されてきた方々に心から敬意を表したいと思います。
